# Market Pulse: Research Questionnaire

Use this before the project week to research the ideas we will use.

For each question, look things up before answering. Short answers are fine, but write them in your own words.

For each section, try to bring back:

- one useful thing you learned
- one link or source that helped
- one question you still have

## Python and Project Setup

1. What is a virtual environment in Python, and why might a project use one?
2. What is a Python package, and how do you install one?
3. What is the command line useful for when running Python projects?

## Data

4. What is a CSV file, and why is it useful for storing rows of data?
5. In Python, how could you load a table of news headlines and show only one company?
6. What might make real-world news data messy or difficult to work with?

## AI Models and APIs

7. What is an API?
8. What is an API key, and why should it be kept private?
9. What is an environment variable, and how can it help keep secrets out of code?
10. What information do you think we send to an AI model when asking it to analyse a headline?
11. What makes a prompt clear or unclear?

## Structured Output

12. What is JSON?
13. Why might it be better for an AI model to return JSON instead of a paragraph?
14. What could go wrong if a model does not return the format your code expects?

## Building the App

15. What is Streamlit, and what kinds of Python projects is it useful for?
16. What is the difference between a Python script and an app someone can interact with?
17. If you were browsing company news, what would you want the app to show first?

## Thinking Like a Builder

18. What mistakes might an AI model make when reading financial news?
19. What should a good 3-minute demo include?
20. What is one part of this project you are most interested in learning?
